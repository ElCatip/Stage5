IOS

iOS (originally iPhone OS) is a mobile operating system for Apple hardware.The user interface of iOS is based on the concept of direct manipulation, using multi-touch gestures. Interface control elements consist of sliders, switches, and buttons. iOS shares with OS X some frameworks such as Core Foundation and Foundation; however, its UI toolkit is Cocoa Touch rather than OS X's Cocoa, so that it provides the UIKit framework rather than the AppKit framework. The latest version IOS9 SDK includes new APIs and services offering multitasking, gaming and support for other applications. IOS 9

XCODE

Software Development kit that allows developers to make applications for the iPhone and iPod Touch, as well as test them in an "iPhone simulator". Use together with Swift to create software programs called apps that can be installed into Iphone and other Apple products. Xcode is an integrated development environment (IDE) containing a suite of software development tools developed by Apple for developing software for OS X and iOS. Xcode also includes Apple's WebObjects tools and frameworks for building Java web applications and web services. The Xcode suite also includes most of Apple's developer documentation, and built-in Interface Builder, an application used to construct graphical user interfaces. This is different from most programming environment because of the graphical user capability which takes awhile to get accustomed to. Besides the simulator, it also has the playground where snippets of code can be tested before running the actual program, displaying the result along with the program code. XCODE reference

Swift

Swift is a new programming language for iOS, OS X, and watchOS apps that builds on the best of C and Objective-C, without the constraints of C compatibility. Like C, Swift uses variables to store and refer to values by an identifying name. Swift also makes extensive use of variables whose values cannot be changed known as constants.Swift also introduces optional types, which handle the absence of a value. This available through us of "let" instead of "var" declarations.In Swift, errors are represented by values of types that conform to the ErrorType protocol. It has autocomplete feature which provides suggestions for parameters to commands. Swift reference
